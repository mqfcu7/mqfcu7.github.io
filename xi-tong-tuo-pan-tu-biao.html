wxPython

wxPython是wxWidgets的python绑定。 wxWidgets 是一系列C++库，它遵循多平台GUI开发框架的设计规则。它有类似于MFC易于使用的API。把它和特定的库链接并编译，可是你的应用程序与目标平台的界面相似。wxWidgets 是一个非常完整的框架，它几乎为你任何需求提供解决方案，并简化使用习惯。 官网：http://www.wxpython.org/ 文档：http://docs.wxwidgets.org/2.8.4/wx_contents.html

1. Hello World

#!/usr/bin/env python
import wx

app = wx.App(False)  # Create a new app, don't redirect stdout/stderr to a window.
frame = wx.Frame(None, wx.ID_ANY, "Hello World") # A Frame is a top-level window.
frame.Show(True)     # Show the frame.
app.MainLoop()
wx.App(False)
>>每一个wxPython程序都是wx.App的一个实例。当需要更复杂的功能是需要扩展wx.App类
>>False表示不将stdout和stderr重定向到一个窗口
wx.Frame(None,wx.ID_ANY,"Hello")
>>一个wx.Frame是一个顶层窗口。
>>None表示没有父元素，wx.ID_ANY表示由wxWidgets自动选择ID，“Hello”为标题
frame.Show(True)
>>显示窗口
app.MainLoop()
>>开始主循环，处理各个事件

2. 事件

所有事件都是一个EVT_*对象，它指明了事件类型。事件处理回调函数需要被绑定到事件。

EvtHandler::Bind(event, handler, source=None, id=wx.ID_ANY, id2=wx.ID_ANY)
>>bind将事件对象和回调函数进行绑定
>>bind是EvtHandle类的一个方法，该类被大多数类继承，因此可以直接调用bind方法
>>event即EVT_*对象，表示事件类型。handle即事件处理回调函数。source用来分辨触发相同事件的不同部件。当有许多按钮、菜单等时，用id来分辨他们。id2用来绑定范围内的所有id。
Callback(self, event)
>>回调函数原型
>>event是wx.Event的子类的一个实例

3. MenuItem类

wxMenuItem::wxMenuItem(wxMenu parentMenu = NULL, int id = wxID_SEPARATOR, const wxString& text = "", const wxString& helpString = "", wxItemKind kind = wxITEM_NORMAL, wxMenu subMenu = NULL)
>>构造函数，创建一个菜单项
>>parentMenu为该菜单项隶属的菜单。id为为菜单项的标示符，默认 wxID_SEPARATOR表示该项为一个分隔符。text为菜单项内容，默认为空。helpString为状态栏显示的帮助内容，默认为空。kind为菜单项样式，可以为单选、复选，默认为正常。subMenu为子菜单，默认无。
>>id可由wx.NewId()产生一个不重复的id
wxMenuItem::Enable(bool enable = true)
>>使能 一个菜单项。若菜单项被disable，则显示变灰且无法响应事件。

wxMenu::wxMenu(const wxString& title = "", long style = 0)

wxMenu::Append(wxMenuItem* menuItem)

wxMenu::AppendSeparator()

5. TaskBarIcon类

该类负责在系统托盘上显示图标，并处理相关事件。

TaskBarIcon::SetIcon(self, icon, tooltip)
>>设置系统托盘的图标
>>icon为图标，可由位图文件转换，见下。tooltip为鼠标在图标上悬停时显示的文字。
>>`img = wx.Bitmap(path)`
>>`icon = wx.IconFromBitmap(img)`
TaskBarIcon::CreatePopupMenu(self)
>>生成右键菜单，继承的类必须重写该方法以定制右键菜单。
>>该方法必选返回一个menu实例。
>>**每次单击右键，TaskBarIcon类都会调用该方法生成一个新的menu。因此保存menu变量是无效的，对menuItem的使能也必须在该方法内完成。**
